**Luvian is the MBSE platform that systems engineers and software developers can finally agree on** — browser-native, real-time collaborative, SysML v2-first, and AI-native from day one.

## The Problem
MBSE adoption is stuck at ~5% of engineering teams because incumbent tools are 1990s-era Java desktop apps (Cameo: 9 GB RAM, hostile UX, SysML v1-locked). Regulated industries — automotive (ISO 26262), aerospace (DO-178C), defense (MIL-STD-882E) — *need* MBSE the most and are served the worst.

> *"Organizations buy the best MBSE tool, but after three months no one was using it."* — one-sys.eu

## Why Now
**Three forces converging for the first time in 30 years:**
1. **SysML v2** (OMG, 2024) breaks every incumbent's data model — multi-year migration window opens.
2. **LLMs production-ready** for structured authoring — collapses the "modeling tax" that capped adoption.
3. **Browser-first migration** — every adjacent engineering tool (CAD, EDA, ALM) has moved; MBSE is the last holdout.

## The Product
- **Browser-native, real-time collaborative canvas** for SysML v2 (Yjs CRDT, no install).
- **AI-native authoring** — ambient suggestions, domain agents, RAG-grounded validation.
- **Progressive maturity gates** (Integration Readiness Reviews, conformance scoring) — tells teams when a model is ready.
- **Bidirectional code ↔ model** — SE and SW engineers consume the same model.
- **Multi-persona views** — KPIs, safety dashboards (HARA / TARA / STPA / SOTIF / FMEA), validation reports.

**Status:** ~780 source files, 42 components, 98 modules. Built on React 19 + Vite, React Flow v12, Yjs/Hocuspocus, Anthropic, Auth0, Supabase. ~40% of capability live; the rest gated behind flags awaiting design-partner feedback.
